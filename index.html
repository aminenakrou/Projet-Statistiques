01 — Introduction

Contexte et Objectifs

Dans le secteur financier, anticiper le risque d'impayé est une nécessité stratégique. Ce projet construit, compare et évalue plusieurs modèles de scoring sur données réelles.

🎯

Problème métier

Identifier les clients susceptibles de ne pas rembourser leur crédit avant l'octroi du prêt. Un faux négatif (impayé non détecté) coûte bien plus qu'un faux positif (refus injustifié).

⚖️

Déséquilibre des classes

Le jeu de données présente 86.7% d'Impayés et seulement 13.3% de Payeurs. Sans correction, tout modèle naïf prédit "Impayé" avec une accuracy illusoire.

📐

Métrique prioritaire

La courbe ROC et l'AUC (Area Under the Curve) permettent une évaluation indépendante du seuil de décision — indispensable pour le risque de crédit.

🔁

Pipeline complet

Exploration → Feature Engineering → SMOTE → Modélisation (LR, Arbre, RF) → Évaluation ROC → Seuil de Youden optimal.

1

Chargement & Nettoyage

Import du fichier pipe-séparé, suppression doublons, encodage des facteurs (Telephone, Assurance, Client, Nb_emprunteurs, Payeur).

2

Analyse Exploratoire (EDA)

Boxplots des variables numériques, barplots conditionnels pour les variables catégorielles, matrice de corrélation.

3

Feature Engineering

Création du Taux d'endettement, de la Mensualité (formule annuités) et du Reste à vivre — trois variables financières hautement discriminantes.

4

Split Train / Test — 70% / 30%

Graine fixée à 42 pour la reproductibilité. Le jeu test reste intact (sans SMOTE) pour une évaluation honnête.

5

SMOTE — Rééquilibrage

Génération de dup_size = 4 copies synthétiques de la classe minoritaire (Payeurs) avec bruit gaussien.

6

Modélisation & Évaluation ROC

3 modèles × 2 versions (sans/avec SMOTE) = 6 configurations comparées via AUC, rappel, F1-Score et seuil de Youden.

02 — Données & EDA

Jeu de Données & Analyse Exploratoire

Fichier GV_RSQ_data_scoring_risque.txt — format pipe-séparé (|). Chaque ligne représente un dossier de crédit anonymisé.

💰

Variables financières

  • Montant du crédit (€)
  • Durée (mois)
  • Taux d'intérêt (%)
  • Ressources client (€)
  • Charges client (€)
🏷️

Variables catégorielles

  • Téléphone (Oui/Non)
  • Assurance (Oui/Non)
  • Client (Ancien/Nouveau)
  • Nb_emprunteurs (facteur)
🎯

Variable cible

Payeur : Impayé (0) — classe majoritaire à 86.7%
Payeur (1) — classe minoritaire à 13.3%

Variables dérivées — Feature Engineering

Taux d'endettement
$$\text{Taux}_{end} = \frac{\text{Charges}}{\text{Ressources}} \times 100$$
Mensualité (annuités)
$$M = \frac{C \cdot \frac{r}{12}}{1 - \left(1 + \frac{r}{12}\right)^{-n}}$$
Reste à vivre
$$\text{RAV} = \text{Ressources} - \text{Charges} - M$$

Avec $C$ = montant, $r$ = taux annuel, $n$ = durée en mois. Ces trois indicateurs sont fortement discriminants : les clients Impayés présentent un taux d'endettement plus élevé et un reste à vivre plus faible.

⚠
Déséquilibre critique : 86.7% des observations sont de classe "Impayé". Un modèle naïf qui prédit "Impayé" systématiquement atteindrait 86.7% d'accuracy mais un rappel de 0% sur les Payeurs — inutilisable en pratique.
03 — Rééquilibrage

SMOTE — Synthetic Minority Over-sampling Technique

SMOTE génère des exemples synthétiques pour la classe minoritaire en interpolant entre observations existantes, plutôt qu'en dupliquant simplement des exemples.

Principe mathématique

Formule SMOTE
$$x_{new} = x_i + \lambda \cdot (x_{voisin} - x_i), \quad \lambda \sim \mathcal{U}(0,1)$$

On choisit un exemple $x_i$ de la classe minoritaire, on sélectionne aléatoirement un de ses $k$ plus proches voisins $x_{voisin}$, puis on génère $x_{new}$ sur le segment les reliant. Dans cette implémentation, le bruit gaussien simule $\lambda$.

Variante implémentée
$$x_{var}^{(d)} = x_{payeur} + \varepsilon, \quad \varepsilon \sim \mathcal{N}(0, \sigma^2)$$

Avec dup_size = 4 itérations → 5 fois plus de Payeurs qu'initialement.

📚
Référence originale : Chawla et al. (2002) — SMOTE: Synthetic Minority Over-sampling Technique, Journal of Artificial Intelligence Research, 16, 321–357.

Impact sur la distribution

Avant SMOTE

Impayé (majoritaire)86.7%
Payeur (minoritaire)13.3%

Après SMOTE (dup_size=4)

Impayé≈ 60.5%
Payeur (5× plus)≈ 39.5%

Optimisation de dup_size

Une grille de dup_size de 1 à 20 est testée sur le F1-Score. La valeur optimale maximise la détection des vrais Payeurs sans dégrader la précision. Le test set reste toujours non augmenté (évaluation honnête).

04 — Modélisation

Les 3 Modèles de Classification

Chaque modèle est entraîné deux fois : une version de référence (sans SMOTE) et une version rééquilibrée (avec SMOTE). Cela donne 6 configurations évaluées sur le même test set.

01

Régression Logistique

Modèle linéaire généralisé

Formulation

$$P(Y=1|X) = \frac{1}{1+e^{-(\beta_0 + \boldsymbol{\beta}^\top \mathbf{X})}}$$

Description

Modèle de référence en scoring de crédit. Estime la probabilité d'impayé via une fonction sigmoïde appliquée à une combinaison linéaire des variables. Les coefficients $\beta_j$ sont interprétables comme des log-odds ratios.

Avantages

  • Interprétabilité totale des coefficients
  • Sortie probabiliste directe
  • Base réglementaire (Bâle II/III)
  • Estimation par maximum de vraisemblance
📖 Hosmer, Lemeshow & Sturdivant (2013) — Applied Logistic Regression, 3rd ed., Wiley.
02

Arbre de Décision

CART — rpart

Critère de partition

$$\text{Gini}(t) = 1 - \sum_{k} p_k^2(t)$$

Description

Algorithme CART (Classification And Regression Trees). Partitionne récursivement l'espace des features selon la règle qui minimise l'impureté de Gini. Le paramètre cp = 0.01 contrôle l'élagage (pruning).

Avantages

  • Visualisation arborescente intuitive
  • Capture les interactions non-linéaires
  • Aucune hypothèse de distribution
  • Sélection implicite de variables
📖 Breiman, Friedman, Stone & Olshen (1984) — Classification and Regression Trees, CRC Press.
03

Forêt Aléatoire

Ensemble — 200 arbres

Prédiction agrégée

$$\hat{f}(x) = \frac{1}{B}\sum_{b=1}^{B} T_b(x, \Theta_b)$$

Description

Construit $B = 200$ arbres de décision sur des sous-échantillons bootstrap. Chaque nœud teste un sous-ensemble aléatoire de $\sqrt{p}$ variables. L'agrégation réduit la variance sans augmenter le biais.

Avantages

  • Meilleure generalisation (out-of-bag error)
  • Robuste aux valeurs aberrantes
  • Importance des variables (Gini decrease)
  • Peu de paramètres à régler
📖 Breiman (2001) — Random Forests, Machine Learning, 45(1), 5–32.

Métriques d'évaluation — Formules

Accuracy
$$\text{Acc} = \frac{VP + VN}{VP + VN + FP + FN}$$
Précision (Positive Predictive Value)
$$\text{Précision} = \frac{VP}{VP + FP}$$
Rappel (Sensibilité / Recall)
$$\text{Rappel} = \frac{VP}{VP + FN}$$
F1-Score (moyenne harmonique)
$$F_1 = 2 \cdot \frac{\text{Précision} \times \text{Rappel}}{\text{Précision} + \text{Rappel}}$$
🔑
En risque de crédit, le rappel est la métrique prioritaire : manquer un vrai impayé (FN) coûte bien plus qu'un refus injustifié (FP). L'accuracy seule est trompeuse sur des données déséquilibrées.
05 — Évaluation

Courbe ROC & AUC

La courbe ROC (Receiver Operating Characteristic) représente le compromis entre Sensibilité et Spécificité pour tous les seuils de décision possibles.

Définitions fondamentales

Taux de Vrais Positifs (TVP / Sensibilité)
$$\text{TVP} = \frac{VP}{VP + FN} = P(\hat{Y}=1 \mid Y=1)$$
Taux de Faux Positifs (TFP)
$$\text{TFP} = \frac{FP}{FP + VN} = 1 - \text{Spécificité}$$
AUC — Area Under the Curve
$$\text{AUC} = \int_0^1 \text{TVP}(\text{TFP}) \, d(\text{TFP}) = P(\hat{p}_1 > \hat{p}_0)$$

L'AUC s'interprète comme la probabilité que le modèle attribue un score plus élevé à un vrai Payeur qu'à un vrai Impayé, choisis aléatoirement. Un modèle parfait atteint AUC = 1, un modèle aléatoire AUC = 0.5.

AUCInterprétation
0.5Modèle aléatoire — aucune discrimination
0.60 – 0.70Discrimination faible
0.70 – 0.80Discrimination acceptable
0.80 – 0.90Bonne discrimination
0.90 – 1.00Excellente discrimination

Courbe ROC — Représentation schématique

Seuil Optimal — Indice de Youden

Indice de Youden
$$J = \max_{\tau}\left[\text{Sensibilité}(\tau) + \text{Spécificité}(\tau) - 1\right]$$

Le seuil $\tau^* = \arg\max J$ est le point sur la courbe ROC le plus éloigné de la diagonale aléatoire. Il correspond au meilleur compromis Sensibilité / Spécificité, sans hypothèse sur le coût relatif des erreurs.

✓
Dans ce projet, le seuil optimal est calculé sur la Forêt Aléatoire avec SMOTE (meilleur AUC) via la fonction coords(roc, "best", best.method = "youden") du package pROC.
📚
Référence méthodologique : Fawcett, T. (2006) — An introduction to ROC analysis, Pattern Recognition Letters, 27(8), 861–874. | Package R : Xavier Robin et al. — pROC: an open-source package for R and S+

Synthèse & Recommandations

Sans SMOTE

Tous les modèles obtiennent une accuracy trompeuse (~87%). Le rappel sur la classe Payeur est très faible : les modèles prédisent "Impayé" pour presque tous les clients.

Avec SMOTE

SMOTE améliore significativement le Rappel et le F1-Score sur les trois modèles. La Forêt Aléatoire en bénéficie le plus grâce à sa capacité à apprendre les frontières complexes.

Modèle recommandé

Forêt Aléatoire + SMOTE — meilleure AUC, meilleur rappel, courbe ROC dominante. Appliquer le seuil de Youden plutôt que 0.5 pour calibrer le compromis risque/refus.
